Teoría

¿Qué es la Programación Orientada a Objetos (POO)?

La Programación Orientada a Objetos (POO) es un paradigma de programación basado en el concepto de "objetos", que pueden contener datos en forma de atributos y código en forma de métodos. Los objetos interactúan entre sí para realizar tareas y resolver problemas.

Instrucciones básicas para trabajar con JSON en JavaScript

JSON (JavaScript Object Notation) es un formato ligero para el intercambio de datos. En JavaScript, se utiliza JSON.stringify() para convertir un objeto a una cadena JSON y JSON.parse() para convertir una cadena JSON a un objeto.

// Convertir un objeto a JSON
const obj = { nombre: 'Juan', edad: 30 };
const jsonString = JSON.stringify(obj);

// Convertir una cadena JSON a objeto
const parsedObj = JSON.parse(jsonString);

¿Cuáles son los pilares de la POO?

  • Encapsulamiento: Oculta los detalles internos del objeto y solo expone una interfaz pública.
  • Abstracción: Representa los conceptos esenciales sin incluir detalles innecesarios.
  • Herencia: Permite crear nuevas clases basadas en clases existentes.
  • Polimorfismo: Permite que un objeto se comporte de diferentes maneras según su tipo.

¿Qué es el polimorfismo? Tipos de polimorfismo

El polimorfismo es una característica que permite que una función o método tenga múltiples formas. Existen dos tipos principales:

  • Polimorfismo en tiempo de compilación: Logrado mediante la sobrecarga de métodos.
  • Polimorfismo en tiempo de ejecución: Logrado mediante herencia e interfaces.

Relaciones entre clases

Las relaciones entre clases se representan en diagramas UML mediante conectores:

  • Asociación: Relación entre dos clases.
  • Agregación: Relación de "todo-parte".
  • Composición: Ciclo de vida de la parte controlado por el todo.
  • Herencia: Relación de "es un".
  • Relación simple: Relación general sin jerarquía.

Definiciones

Clase:
Plantilla para crear objetos que define atributos y métodos.
Instancia:
Un objeto creado a partir de una clase.
Atributos:
Variables que definen el estado de un objeto.
Métodos:
Funciones que definen el comportamiento de un objeto.
Herencia:
Mecanismo por el cual una clase hereda atributos y métodos de otra.
Visibilidad:
Especifica el acceso permitido a los miembros de una clase.
Abstracción:
Proceso de simplificar una entidad al ocultar detalles complejos.
Objeto:
Instancia de una clase que encapsula datos y métodos.
Encapsulamiento:
Principio de ocultar los detalles internos de un objeto.
JSON:
Formato ligero de intercambio de datos fácil de leer y escribir.
